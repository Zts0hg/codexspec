## Phase 2: 测试验证

> **注意**: 以下测试任务为手动测试，需要在 Claude Code 环境中由用户执行验证。

### Task 2.1: 测试主分支检测提示 ⏳

- **类型**: 测试
- **文件**: 无（手动测试）
- **描述**: 在 main 分支执行 `/codexspec:specify "测试功能"`，验证分支检测提示正确显示
- **依赖**: Task 1.1
- **复杂度**: 低
- **状态**: 待用户验证
- **测试步骤**:
  1. 切换到 main 分支
  2. 执行 `/codexspec:specify "测试功能"`
  3. 验证显示"您当前在主分支「main」上"提示
  4. 验证提供 3 个选项

### Task 2.2: 测试创建新分支流程 ⏳

- **类型**: 测试
- **文件**: 无（手动测试）
- **描述**: 选择"创建新功能分支"，验证分支创建成功
- **依赖**: Task 2.1
- **复杂度**: 低
- **状态**: 待用户验证
- **测试步骤**:
  1. 在主分支检测提示中选择"创建新功能分支"
  2. 输入功能名称 "test-feature"
  3. 验证新分支创建成功（格式：`YYYY-MMDD-HHMMxx-test-feature`）
  4. 验证自动切换到新分支
  5. 验证命令继续执行

### Task 2.3: 测试跳过分支创建 ⏳

- **类型**: 测试
- **文件**: 无（手动测试）
- **描述**: 选择"在当前分支继续"，验证命令继续执行
- **依赖**: Task 2.1
- **复杂度**: 低
- **状态**: 待用户验证
- **测试步骤**:
  1. 在主分支检测提示中选择"在当前分支继续"
  2. 验证不创建新分支
  3. 验证命令正常继续执行

### Task 2.4: 测试功能分支无提示 [P] ⏳

- **类型**: 测试
- **文件**: 无（手动测试）
- **描述**: 在功能分支执行命令，验证不触发分支检测提示
- **依赖**: Task 1.1
- **复杂度**: 低
- **状态**: 待用户验证
- **测试步骤**:
  1. 切换到功能分支（如 `2026-0321-1430k7-user-auth`）
  2. 执行 `/codexspec:specify "测试功能"`
  3. 验证不显示分支检测提示
  4. 验证命令直接执行

### Task 2.5: 测试非 Git 环境处理 [P] ⏳

- **类型**: 测试
- **文件**: 无（手动测试）
- **描述**: 在非 Git 目录执行命令，验证跳过分支检测
- **依赖**: Task 1.1
- **复杂度**: 低
- **状态**: 待用户验证
- **测试步骤**:
  1. 切换到非 Git 仓库目录（如 `/tmp`）
  2. 执行 `/codexspec:specify "测试功能"`
  3. 验证跳过分支检测
  4. 验证命令正常执行（不报错）
