## Resolved During Auto-Fix

### MINOR-001: T004 git-diff scope check did not account for script regeneration

- **Evidence**: `codexspec init --force` "Overwrite regenerable files (commands/scripts)" — `src/codexspec/__init__.py:510-521`. T003 runs exactly this command.
- **Location**: `tasks.md` → T004, SC-003 / REQ-007 bullet.
- **Mismatch**: T004 expected `git diff` to show only the 2 source templates + 4 derived command/skill files. But `--force` also regenerates `.codexspec/scripts/`; if those scripts were stale, they would appear in the diff and falsely fail T004's scope check.
- **Impact**: False-positive verification failure if `.codexspec/scripts/` is out of sync, confusing the implementer and blocking on a non-defect.
- **Remediation applied (round 1)**: Reworded T004's SC-003 bullet to distinguish the real intent (no OTHER template's prose changed) from expected non-template changes (4 derived files + possibly `.codexspec/scripts/`), explicitly noting script changes are expected regeneration output. Supported by the verified `--force` behavior; no new decision.
- **Round-2 verification**: Resolved. T004's scope check is now robust to script regeneration.
