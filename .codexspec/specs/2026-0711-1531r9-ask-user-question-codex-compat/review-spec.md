## Risk Advisories

### RA-001: Assumption 1 — Codex agent may not invoke `request_user_input`

- **Applicability condition**: Under the DEC-002 wording, the Codex agent is expected to recognize `request_user_input` as its native structured-question tool and call it.
- **Concrete risk**: If the Codex GPT-5 agent misreads the "e.g., `AskUserQuestion` or `request_user_input`" instruction as a prose question, the structured-choice intent is lost and the user gets a plain-text question instead.
- **Relationship to confirmed goal**: Directly affects NEED-001's success under Codex. The user accepted this risk by choosing the "name the tools" mechanism over pure neutral wording.
- **Suggested follow-up (not auto-fixable)**: If manual verification (OPEN-002) shows the Codex agent does not invoke `request_user_input`, consider escalating to a more directive instruction or re-evaluating DEC-002. This is a future user decision, not a current defect.
