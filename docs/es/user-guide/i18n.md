### Establecer el idioma

#### Durante la inicializaciĂłn

```bash

# Totalmente no interactivo: base zh-CN, mensajes de commit en inglĂŠs

# Establecer cada dimensiĂłn explĂ­citamente (scriptable, sin prompts)
```

La primera inicializaciĂłn en una TTY sin `--lang` (y sin las tres flags de dimensiĂłn) solicita un idioma base; en un entorno sin TTY (CI/scripts) el valor predeterminado es `en`. Volver a ejecutar `init` preserva cualquier clave de idioma que no hayas especificado.

#### DespuĂŠs de la inicializaciĂłn

```bash

# Cambiar una Ăşnica dimensiĂłn
```

## i18n de la documentaciĂłn en GitHub Pages

> **Idiomas del sitio de documentaciĂłn frente a idiomas de la CLI/tu tiempo de ejecuciĂłn: no los confundas.**
>
> - **Sitio de documentaciĂłn (esta secciĂłn): 8 idiomas.** El sitio de GitHub Pages construido por MkDocs publica exactamente los ocho locales enumerados a continuaciĂłn (`en`, `zh`, `ja`, `ko`, `es`, `fr`, `de`, `pt-BR`). > - **CLI / tiempo de ejecuciĂłn (la secciĂłn anterior): 13 idiomas.** Las dimensiones `language.*` que configuras en `.codexspec/config.yml` aceptan 13 cĂłdigos soportados (`en`, `zh-CN`, `zh-TW`, `ja`, `ko`, `es`, `fr`, `de`, `pt-BR`, `ru`, `it`, `ar`, `hi`), traducidos al vuelo por el LLM. Consulta la tabla "Idiomas soportados" en el [README](https://github.com/Zts0hg/codexspec#internationalization-i18n) del proyecto para la lista canĂłnica.
>
> Ambos conjuntos se solapan, pero no son idĂŠnticos: el sitio de documentaciĂłn no tiene los locales `zh-TW`/`ru`/`it`/`ar`/`hi`, y la CLI no usa los cĂłdigos `zh`/`de`/... sin prefijo que MkDocs emplea como nombres de carpeta.

### Traducir la documentaciĂłn (solo para mantenedores)

> **Nota**: Los comandos `/codexspec:translate-docs` y `/codexspec:check-i18n-semantics` son **herramientas exclusivas para mantenedores de CodexSpec**. EstĂĄn fuertemente acoplados al layout i18n de MkDocs de este repositorio (`docs/{lang}/`) y al glosario interno en `docs/i18n/glossary.yml`. **No** se distribuyen a proyectos de usuario mediante `codexspec init` y no estĂĄn disponibles como slash commands para el usuario final.
>
> Si trabajas en el propio CodexSpec, estos comandos viven en `.claude/commands/codexspec/` y se invocan igual que cualquier otro slash command. Los usuarios finales que traduzcan la documentaciĂłn de su propio proyecto deben usar Claude directamente, con el flujo de trabajo que prefieran.

Para los mantenedores de CodexSpec, usa el comando `/codexspec:translate-docs` para traducir la documentaciĂłn:

```bash

/codexspec:translate-docs --lang zh

# Traducir un archivo especĂ­fico al japonĂŠs
/codexspec:translate-docs user-guide/installation.md --lang ja

# TraducciĂłn incremental (solo archivos modificados)
```

### MetodologĂ­a de traducciĂłn

Las traducciones pretenden leerse de forma natural en tu idioma, no como reflejos palabra a palabra del inglĂŠs:

- **El sentido, primero.** Cada pasaje se traduce a partir de lo que comunica y luego se reexpresa con la redacciĂłn y la estructura oracional propias del idioma destino.
- **Los tĂŠrminos en inglĂŠs se mantienen cuando son mĂĄs claros.** Los tĂŠrminos tĂŠcnicos sin un equivalente nativo adecuado âo mĂĄs reconocibles en inglĂŠsâ se conservan en inglĂŠs en lugar de traducirse de forma forzada. Por ejemplo, "AI coding agent" se convierte en una frase nativa natural como "AI çźç¨ĺŠć", no en una traducciĂłn literal y rĂ­gida.
- **Fluidez nativa.** El objetivo es una prosa que se lea como si hubiera sido escrita originalmente en tu idioma.

### IntegraciĂłn CI/CD

El flujo de trabajo `.github/workflows/docs-i18n.yml` es **Ăşnicamente una puerta de compilaciĂłn (build gate)**: en cada cambio de `docs/**` / `mkdocs.yml` (push a `main` y pull requests) ejecuta `mkdocs build --strict` en todas las versiones de idioma para detectar errores de compilaciĂłn, enlaces rotos y contenido mal formado. **No** traduce nada.

Las traducciones se producen y se confirman manualmente mediante `/codexspec:translate-docs` (vĂŠase mĂĄs arriba), con la fuente en inglĂŠs y todas las traducciones en un Ăşnico commit revisado y atĂłmico. La traducciĂłn automĂĄtica en CI se eliminĂł de forma intencional: el job anterior nunca llegaba a ejecutarse realmente (un `if` mal formado lo saltaba), requerĂ­a una clave API permanente y habrĂ­a confirmado traducciones sin revisar directamente a `main`.
